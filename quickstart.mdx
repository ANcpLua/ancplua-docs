# Quickstart

This guide walks you through adding ANcpLua.NET.Sdk to your .NET projects.

## Prerequisites

- .NET SDK 8.0 or later
- Central Package Management enabled

If you do not have Central Package Management, create a `Directory.Packages.props` file in your repository root:

```xml
<Project>
  <PropertyGroup>
    <ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>
  </PropertyGroup>
</Project>
```

## Step 1: Add SDK to global.json

Create or update your `global.json` file in the repository root. Check [NuGet](https://www.nuget.org/packages/ANcpLua.NET.Sdk/) for the latest version.

```json
{
  "msbuild-sdks": {
    "ANcpLua.NET.Sdk": "1.3.31"
  }
}
```

<Tip>
Use [Dependabot](https://docs.github.com/en/code-security/dependabot) or [Renovate](https://docs.renovatebot.com/) to automatically update SDK versions.
</Tip>

## Step 2: Update Your Project File

Replace the SDK attribute in your `.csproj` file:

<Tabs>
  <Tab title="Library / Console">
    ```xml
    <Project Sdk="ANcpLua.NET.Sdk">
      <PropertyGroup>
        <TargetFramework>net10.0</TargetFramework>
      </PropertyGroup>
    </Project>
    ```
  </Tab>
  <Tab title="Web API">
    ```xml
    <Project Sdk="ANcpLua.NET.Sdk.Web">
      <PropertyGroup>
        <TargetFramework>net10.0</TargetFramework>
      </PropertyGroup>
    </Project>
    ```
  </Tab>
  <Tab title="Test Project">
    ```xml
    <Project Sdk="ANcpLua.NET.Sdk.Test">
      <PropertyGroup>
        <TargetFramework>net10.0</TargetFramework>
      </PropertyGroup>
    </Project>
    ```
  </Tab>
</Tabs>

## Step 3: Build

Run `dotnet build` to verify the SDK is working:

```bash
dotnet build
```

The SDK will automatically:

- Enable nullable reference types
- Set LangVersion to `latest`
- Add ANcpLua.Analyzers for code quality enforcement
- Configure banned API analyzers
- Enable deterministic builds

## Project Type Examples

### Class Library

```xml
<Project Sdk="ANcpLua.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net10.0</TargetFramework>
    <IsPackable>true</IsPackable>
    <PackageId>MyCompany.MyLibrary</PackageId>
  </PropertyGroup>
</Project>
```

### ASP.NET Core Web API

```xml
<Project Sdk="ANcpLua.NET.Sdk.Web">
  <PropertyGroup>
    <TargetFramework>net10.0</TargetFramework>
  </PropertyGroup>
</Project>
```

The Web SDK adds:

- OpenTelemetry integration with OTLP export
- Health check endpoints at `/health` and `/alive`
- HTTP client resilience (retries, circuit breakers)
- DevLogs for browser console capture

### xUnit Test Project

```xml
<Project Sdk="ANcpLua.NET.Sdk.Test">
  <PropertyGroup>
    <TargetFramework>net10.0</TargetFramework>
  </PropertyGroup>
</Project>
```

The Test SDK adds:

- xUnit v3 with MTP (Microsoft Testing Platform)
- Test coverage tooling
- FakeLogger extensions (opt-in)

## Multi-targeting Legacy Frameworks

For libraries targeting older frameworks like `netstandard2.0`, the SDK automatically provides polyfills:

```xml
<Project Sdk="ANcpLua.NET.Sdk">
  <PropertyGroup>
    <TargetFrameworks>net10.0;netstandard2.0</TargetFrameworks>
  </PropertyGroup>
</Project>
```

Automatic polyfills include:

- Index and Range operators (`^1`, `..`)
- `IsExternalInit` for init-only setters
- String extensions (`Contains(char)`, `ReplaceLineEndings()`)

See [Polyfills](/sdk/polyfills) for the complete list.
