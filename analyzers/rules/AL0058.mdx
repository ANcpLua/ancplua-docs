## Description

Locking on `this` exposes the synchronization target to any code that holds a reference to the instance, creating deadlock risks.

## Bad Code

```csharp
public void Update()
{
    lock (this) { _count++; }
}
```

## Good Code

```csharp
private readonly Lock _syncRoot = new();
public void Update()
{
    lock (_syncRoot) { _count++; }
}
```
